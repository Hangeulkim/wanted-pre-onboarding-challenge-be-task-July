HashMap의 경우 Key에 대한 해시값을 기반으로 값을 저장하고 조회합니다.
해싱의 결과는 정수여야하며 배열의 크기를 넘어서면 안되기에 인덱스에 (size-1)&hash로 인덱스를 구합니다.
이를 통해 O(1)로 데이터에 접근이 가능하며 Node 객체로 버킷을 만들어 관리합니다.
hash값이 겹칠 경우가 존재하므로 버킷에 데이터가 들어있는 비율이 load_factor를 넘기게 된다면 버킷의 사이즈를 조절합니다.
충돌 수가 적으면 LinkdedList로 충돌된 노드들을 관리합니다.
그래도 충돌이 일어나게 된다면 Red-Black Tree방식을 이용해 데이터를 저장합니다.
